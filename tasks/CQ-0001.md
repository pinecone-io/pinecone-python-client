### CQ-0001: Remove metric-unaware sort from QueryResponse.__post_init__

- **Status:** pending
- **Priority:** 2
- **Claims:** quality
- **Depends on:** none
- **Playbook:** none
- **Context:**
  - `QueryResponse.__post_init__` (pinecone/models/vectors/responses.py:54-60) always sorts
    matches by descending score (`reverse=True`). This is metric-unaware: for euclidean metric,
    lower score = more similar, so ascending order is correct. The Pinecone API already returns
    matches sorted by relevance (API spec: "ordered from most similar to least similar"), so
    the re-sort is redundant for API responses and actively harmful for euclidean because it
    reverses the correct order.
  - The `QueryResultsAggregator` (query_aggregator.py) correctly handles euclidean by sorting
    ascending. This creates a contract mismatch: `query()` and `query_namespaces()` would
    produce different orderings for the same euclidean data.
  - **Fix approach:** Remove the sort from `__post_init__`. Keep the null-namespace normalization.
    The API already returns sorted results, and the aggregator handles its own sorting. If a
    safety sort is desired, it needs the metric as context (which QueryResponse doesn't have).
  - Conventions: `docs/conventions/types.md`
- **Verify:** `cd sdks/python-sdk2 && uv run mypy --strict pinecone/ && uv run pytest tests/unit/ -x -v -k "query"`
